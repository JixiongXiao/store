// DNS:Domain Name Server
// DNS解析就是将域名解析成IP地址
// 特征：NDS中保存了一张域名与对应的IP地址表，一个域名对应一个IP地址，一个IP地址可以对应多个域名

// DNS解析过程 视频26分钟开始
// Search -> DNS本地服务器（就是本地网络运营商）（如果没有缓存）-> 跟服务器 ->.com域服务器 ->域服务器

// IP Internet Protocol Address
// IPv4 作用：分配给用户上网使用的互联网协议。 10进制 比如192.168.0.1 一共有4组（4个字节） 长度32
// IPv6 16进制 (1-10 ABCDEF) 有8组，长度128位 
// IPv6优势 路由表更小，对自动配置支持成功率高 ipv4成功率低 更高的安全性

// IP端口号Port
// 如果IP地址是游乐园 端口号就是不同的娱乐设施
// IP提供了服务器地址，服务地址中有不同的服务，port就是服务的端口号
// 端口号范围0-65535
// 默认端口 http协议下（80），https协议下（443） FTP协议下（20，21）（背下来）
// 打比方
// IP：上海市浦东新区， 域名：上海迪士尼乐园，端口：乐园海盗船入口

// TCP transmission control protocol 传输控制协议 （就像打电话，）
// 特点 面向连接（收发数据前，必须建立可靠的连接）
// 建立连接的基础：三次握手
// 应用场景：数据必须准确无误的收发，HTTP请求，TP文件传输，邮件收发
// 优点：稳定，重传机制，拥塞控制机制，可以实时断开连接
// 缺点：速度慢，效率低，占用资源，容易被攻击(三次握手时容易被攻击， DOS，DDOS攻击)

// TCP/IP协议组 这个协议组制定了数据封装，确定地址，传输，路由，数据接收的标准，提供点对点的连接机制

// UDP User Data Protocol用户数据报协议
// 特点 面向无连接(不可靠的协议，无状态传输机制) 无连接信息发送机制
// 应用场景：毋须确保通讯质量且要求速度快，无需确保信息完整，信息收发，语音通话，直播（QQ）
// 优点：安全，快速，漏洞少（UDP flood攻击）
// 缺点：不可靠，不稳定，容易丢包

// HTTP与HTTPS
// HTTP Hyper Text Transfer Protocol 超文本传输协议
// 定义：客户端和服务器端请求和应答的标准，用于从WEB服务器端传输超文本到本地浏览器的传输协议
// HTTP请求：按照协议规定先向WEB服务器发送的将超文本传输到本地浏览器的请求

// HTTPS Hyper Text Transfer Protocol Secure 超文本传输安全协议
// 定义： HTTP的安全版（安全基础时SSL/TLS）
// SSL：Secure Sockets Layer 安全套接层
// TLS： Transport Layer Security 传输层安全
// 两者一起用 为网络通信提供安全及数据完整性的一种安全协议，对网络连接进行加密。 可以理解为 HTTPS就是HTTP+SSL/TLS

// 两者的差别
// 1.HTTP是不安全的，HTTPS可以防止被攻击
// 2.HTTP协议的传输内容都是明文，直接在TCP连接上运行，客户端和服务器都无法验证对方身份。
// 3.HTTPS协议的传输内容都被SSL/TLS加密，且运行在SSL/TLS上，SSL/TLS运行在TCP连接上，所以数据传输时安全的
// HTTP： 先建立TCP连接，然后发送请求
HTTPS： 先建立TCP连接，然后发送HTTP请求，在这个过程中，传输的内容被SSL/TLS加密
